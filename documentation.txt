# Документация для фронтенд-разработчиков проекта shkolamoskva.space

## Общая структура API

Все API-эндпоинты доступны по базовому URL `https://api.shkolamoskva.space/api/v1/` (или локально `http://localhost:8000/api/v1/`). Все эндпоинты возвращают JSON-ответы.

## Структура разделов сайта

### Главная страница (`/`)

#### 1. Баннеры (бanners)
- **Эндпоинт:** `/banners/`
- **Метод:** `GET`
- **Поля:**
  - `id` (int) — уникальный идентификатор баннера
  - `title` (str) — заголовок баннера
  - `description` (str) — описание баннера
 - `is_active` (bool) — активен ли баннер
  - `image_url` (str) — URL изображения баннера
- **Ошибки:**
  - `500` — внутренняя ошибка сервера
- **Схема ответа:** [`app/api/banners/schemas.py`](app/api/banners/schemas.py:1)

#### 2. Проекты (projects)
- **Эндпоинт:** `/projects/`
- **Метод:** `GET`
- **Поля:**
  - `id` (int) — уникальный идентификатор проекта
  - `title` (str) — заголовок проекта
  - `project_link` (str) — ссылка на проект
  - `keywords` (list[str]) — ключевые слова
  - `short_description` (str) — краткое описание для главной страницы
  - `image_url` (str) — URL изображения проекта
  - `theme` (str) — тема проекта
  - `category` (str) — категория проекта
  - `is_active` (bool) — активен ли проект
- **Ошибки:**
  - `500` — внутренняя ошибка сервера
- **Схема ответа:** [`app/api/projects/schemas.py`](app/api/projects/schemas.py:1)

#### 3. Новости (news)
- **Эндпоинт:** `/news/`
- **Метод:** `GET`
- **Поля:**
  - `id` (int) — уникальный идентификатор новости
  - `title` (str) — заголовок новости
  - `news_link` (str) — ссылка на новость
  - `keywords` (list[str]) — ключевые слова
  - `image_url` (str) — URL изображения новости
  - `short_description` (str) — краткое описание новости
  - `date` (str) — дата новости в формате `dd.mm.YYYY`
  - `news_type` (str) — тип новости
- **Ошибки:**
  - `500` — внутренняя ошибка сервера
- **Схема ответа:** [`app/api/news/schemas.py`](app/api/news/schemas.py:1)

#### 4. Типы новостей для подписки (news types)
- **Эндпоинт:** `/news/types/`
- **Метод:** `GET`
- **Поля:**
  - `type` (str) — тип новости
- **Ошибки:**
  - `500` — внутренняя ошибка сервера

#### 5. Форма обратной связи (feedback)
- **Эндпоинт:** `/feedbacks/`
- **Метод:** `POST`
- **Поля для отправки:**
  - `name` (str) — имя пользователя
  - `phone` (str, optional) — номер телефона
  - `email` (str, optional) — email пользователя
  - `message` (str) — текст сообщения
- **Ошибки:**
  - `422` — ошибка валидации данных
  - `500` — внутренняя ошибка сервера
- **Схема запроса:** [`app/api/feedbacks/schemas.py`](app/api/feedbacks/schemas.py:1)

#### 6. Форма опроса (polls)
- **Эндпоинт:** `/polls/current/`
- **Метод:** `GET`
- **Поля:**
  - `id` (int) — уникальный идентификатор опроса
  - `question` (str) — тема/вопрос опроса
- **Отправка ответа:**
  - **Эндпоинт:** `/polls/answer/`
  - **Метод:** `POST`
  - **Поля для отправки:**
    - `answer` (str) — текст ответа пользователя
- **Ошибки:**
  - `422` — ошибка валидации данных
  - `500` — внутренняя ошибка сервера
- **Схема ответа:** [`app/api/polls/schemas.py`](app/api/polls/schemas.py:1)

#### 7. Контакты (contacts)
- **Эндпоинт:** `/contacts/`
- **Метод:** `GET`
- **Поля:**
  - `id` (int) — уникальный идентификатор контакта
  - `email` (str) — контактный email

## Общие ошибки и коды состояния

- `200` — успешный запрос
- `400` — неверный запрос (ошибка валидации)
- `404` — ресурс не найден
- `422` — ошибка валидации данных
- `500` — внутренняя ошибка сервера

## Общие схемы данных

### Файлы и изображения
Все файлы и изображения возвращаются в виде URL-адресов, по которым можно получить доступ к файлам. Файлы загружаются на сервер и хранятся в системе файлов.

### Даты
Все даты возвращаются в формате `dd.mm.YYYY` или `dd.mm.YYYY hh:mm` в зависимости от контекста.

### Валидация
Все поля проходят строгую валидацию на стороне сервера. Невалидные данные будут возвращать ошибку `422` с описанием проблемы.

## Админ-панель

Все сущности, описанные выше, могут быть управляться через админ-панель (реализация которой будет добавлена позже). Через админ-панель можно:
- Создавать, обновлять, удалять сущности
- Активировать/деактивировать элементы (где применимо)
- Отвечать на формы обратной связи и вопросы пользователей
- Управлять файлами и изображениями
- Просматривать и анализировать ответы на опросы
